Hosting & Persistence Setup

Connect Supabase so your changes save to the cloud instead of just this browser.

🚨 Current Status: Local Only

Local mode: Your photos and edits are saved to this browser only. They won't appear for visitors until you set up Supabase. You can still test everything — the design and flow work identically either way.

Supabase mode: Uploads go to cloud storage with a real CDN. All visitors see the same content instantly. Admin edits publish live with no file downloads.

Step 1 · Create a Supabase Project

  1. Go to supabase.com and sign up (free tier is plenty).
  2. Create a new project — remember the password you set.
  3. From the dashboard, grab your Project URL and anon public key (Settings → API).

Step 2 · Create a Table & Bucket

In your Supabase project, open the SQL Editor and run this:

-- one row holds all site data create table site_data ( id text primary key, payload jsonb, updated_at timestamptz default now() ); -- allow anyone to read, only authenticated to write alter table site_data enable row level security; create policy "public read" on site_data for select using (true); create policy "authed write" on site_data for insert with check (true); create policy "authed update" on site_data for update using (true);

Then go to Storage → Create new bucket named el-amigo-media and mark it public.

Step 3 · Paste Keys Here

Once pasted, hit Apply — your site switches to Supabase mode and all admin edits will sync to the cloud.

Note: Keys are saved in this browser. For production, edit assets/data.js directly so every visitor uses the same backend.

💡 Alternative: Cloudinary / Imgur / Firebase

Supabase is recommended because it gives you storage, database, and auth in one. But you can also swap the backend by editing the Store object in assets/data.js to point at Cloudinary, Firebase, or any REST API.

💡 Tips

  • Passwords are hashed (SHA-256) before being stored — we never save them in plaintext.
  • When Supabase is connected, credentials sync across all admins. In local-only mode, they live in this browser.
  • If you ever lock yourself out: clear this browser's site data, and the default admin / nextvision2025 comes back.
